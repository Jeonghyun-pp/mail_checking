# mailchecking

A Snov.io-style outreach platform — built phase by phase.

| Phase | Scope | Status |
|-------|-------|--------|
| 0 | Infra: Postgres, Redis, Prisma, app shell | ✅ Done |
| 1 | Email Finder & Verifier (single + bulk) | ✅ Done |
| 2 | Cold email campaigns & drip sequences | ✅ Done |
| 3 | CRM pipelines, deals, tasks & activity | ✅ Done |
| 4 | Email warm-up & deliverability test | ✅ Done |
| 5 | Public API, Chrome extension, webhooks, CSV | ✅ Done |
| + | Auth, team workspace, MCP server | ✅ Done |

## Stack

- **Next.js 16** (App Router, TypeScript) — UI + REST API
- **PostgreSQL + Prisma** — data layer
- **BullMQ + Redis** — background bulk jobs
- **Nodemailer** — SMTP sending (Phase 2)
- **Tailwind CSS** — UI

## Getting started

```bash
# 1. Start Postgres + Redis + Mailpit (dev SMTP)
docker compose up -d

# 2. Apply the database schema
npm run db:migrate

# 3. Run the app (port 3000) and the background worker — in two terminals
npm run dev
npm run worker
```

Open http://localhost:3000.

## How it works

**Email Verifier** — runs syntax → MX lookup → disposable/role checks → live
SMTP mailbox probe → catch-all detection, then scores deliverability 0-100.
Requires outbound port 25; if blocked, results fall back to `UNKNOWN`.

**Email Finder** — generates candidate addresses from common corporate
patterns (`first.last`, `flast`, …), verifies them, and returns the best
confirmed match (or the most likely guess for catch-all domains).

**Bulk jobs** — verification/finder runs over many rows are queued to Redis
and processed by the worker (`npm run worker`) with bounded concurrency.

**Campaigns** — multi-step drip sequences sent from a connected mailbox.
The worker ticks every 60s, delivers each recipient's due step (with
`{{variable}}` personalization), respects per-mailbox daily limits, and
advances recipients through the sequence. Opens and clicks are tracked via
an injected pixel and link redirects. In development, all mail is caught by
Mailpit at http://localhost:8025.

## Authentication & team workspace

This is a single-team internal tool. Email/password auth (scrypt + HMAC-signed
session cookies); `src/proxy.ts` gates every page and API route (except
`/login`, `/signup`, `/api/auth/*`, `/api/v1/*`, `/api/track/*`).

All teammates share **one workspace** — the same leads, campaigns, CRM and
mailboxes — so data is filed under a shared workspace user, not per-person.
Sign-up requires a single-use **invite code** generated by an admin
(Settings → Team invites). The first-ever account becomes the admin.
Reset a password with `tsx scripts/set-password.ts <email> <password>`.

## MCP server

Teammates can drive the app from their own Claude Code / Codex via the MCP
server at `/api/mcp` (28 tools). See [MCP.md](MCP.md) for setup.
